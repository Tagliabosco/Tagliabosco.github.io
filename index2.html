  1. Förklara kort vad ett element, attribut och värde är för något

    2. Ett element är en komponent/del av ett html-dokument som kan innehålla en eller flera "taggar" (attribut), som tillsammans bygger upp en hemsida. Ett attribut är alltså en modifierare av ett html-element. Dvs ett attribut ger ett element en viss funktion eller modifikation. Ett värde är det uttryck som definierar ett attribut i ett element. Dvs värdet är det som ger attributet dess funktion och unika uttryck.

  2. Förklara vad HTTP är för något och vad ett kommunikationsprotokoll innebär

    3. HTTP eller hypertext transfer protocol är ett kommunikationsprotokoll/applikationsprotokoll som nyttjas för att överföra webbsidor via informationsverktyget WWW (World Wide Web) genom hyperlänkar mellan noder (innehållande text). HTTP kodar och transporterar alltså information (olika former av media) mellan en webbläsare och en webbserver över internet. Ett kommunikationsprotokoll är de regler och strukturer som definierar det format genom vilket data överförs på internet. Det finns många olika sorters kommunikationsprotokoll. Ett känt exempel är IP (Internet Protocol), som definierar hur data skall skickas från en dator till en annan via internet. Enligt detta protokoll så har varje dator en egen så kallad IP-adress som identifierar respektive dator från varandra.

  3. Vilka är W3C och vilken är deras huvudsakliga uppgift?

    4. W3C är ett konsortium och en internationell organisation som, genom deras breda nätverk bestående av bland annat underorganisationer, anställda och allmänheten, arbetar för att ta fram standarder inom webbutveckling. Enligt dem själva är deras övergripande mål att Ta webben till dess fulla potential

  4. Vad står WWW för och vad är detta för något?

    5. WWW står för World Wide Web och är en tjänst som gör tillgänglig en samling hemsidor(koddokument) lagrade i webbservrar, som i sin tur är kopplade till lokala datorer över internet. Dessa webssidor kommunicerar med varandra genom HTML-kod och så kallade hyperlänkar, vilka opereras genom HTTP. Applikationer/program som kallas webbläsare gör dessa lagrade hemsidor åtkomoliga.

  5. En typisk internetadress kan se ut på följande vis http://www.facebook.com/home.php
    • Vad kallas denna internetadress på fackspråk (en förkortning)?

      • URL eller Uniform Resource Locator.

    • Vad betyder denna förkortning?

      • En URL (Uniform Resource Locator) är en sträng/address som identifierar en viss resurs på internet, oftast en sorts hemsida.

    • Vad kallas a http:// b www.facebook.com och c /home.php?

      • a: ett kommunikationsprotokoll. b: värdnamn/hostname (kallas också ett nodnamn). c: Ett filnamn.

  6. Vad använder vi kommunikationsprotokollet FTP till?

    7. FTP eller file transfer protocol används för att skicka data/filer mellan TCP/IP uppkopplingar och datorer(ofta servrar). Dvs från ett system till ett annat. Det är bland annat användbart för överföringar som skall göras i bakgrunden eller filöverföringar som kräver precision, exempelvis backup av data, replikation, moln-tjänster eller webbdelning för att ladda upp/ner data från en klient(en dator) till en server eller vise versa.

  7. Vilken modell bygger HTTP på? Modellen beskriver förhållandet mellan en dator som svarar och en dator som ställer förfrågan om att få se webbplatserna.

    8. HTTP bygger på en så kallad klient-server modell där klienten (en dator, ofta genom ett webbläsarprogram) skickar en förfrågan till en server (en annan dator) om tillgång till en viss data (ofta en hemsida). Servern kör förfrågan genom ett så kallat serverprogram som sedan skickar tillbaka en HTTP till webbläsaren(den första datorn) som svar. Klienten (den förfrågande datorn) processerar sedan svaret från servern och användaren bereds tillgång till datan.

  8. Vad står HTML för och vilket typ av språk är detta?

    9. HTML står för HyperText Markup Language och är ett märkspråk eller ett sidbeskrivningsspråk. Detta ger format till dokument som består av särskild kod (textkod) innehållande olika taggar (element, attribut och värden). Denna kod ger instruktioner till ett program om hur media i dokumentet skall presenteras.

  9. Varför använder vi oss av indentering (indrag) när vi skriver HTML-kod?

    10. Indentering är en teknik som används för att visuellt underlätta för programmeraren i dennes arbete. Bland annat gör indentering det enklare att se hur inheritance(arv) är strukturerat. Därigenom blir koden enklare att arbeta med.

  10. Blir det någon skillnad på utseendet av sidan när du indenterar din HTML-kod?

    11. Nej, indentering är endast för att underlätta för programmerare/webbutvecklare.

  11. Varför tror du att vi lär oss behärska HTML-kod från grunden till skillnad från att generera HTML-kod med hjälp av s.k. WYSIWYG-programvaror och HTML-generatorer?

    12. I vissa fall blid koden snabbare och mer effektiv om man skriver den själv, i andra fall är det tvärtom. Om man inte lär sig HTML från grunden är det svårt att felsöka kod och effektivt kunna arbeta med större projekt. Dessutom lär man sig då aldrig att bygga upp en egen unik hemsida specifikt kodad för ett särskilt ändamål, med särskild design och användargränssnitt.

  12. Vilka två delar består ett HTML-dokument av? Vilken sorts information finns i respektive del?

    13. Ett HTML-dokument består av två delar, dels "huvud" (head) delen som innehåller dokumentets titel och metadata. Sedan har vi "kroppen" (body) delen som innehåller den data som skall presenteras på hemsidan.

  13. Varför har vi s.k. HTML-kommentarer i vår kod?

    14. Detta är för att underlätta arbetet med koden långsiktigt. Antingen om vi själva skrivit kod, och vi inte riktigt minns varför vi skrev koden, eller vilket syfte/funktion koden har, kan kommentarer vara ett bra hjälpmedel. Samma princip kan appliceras på scenariot där vi arbetar med andra programmerare i samma kodbas, där de andra kan behöva arbeta i din kod. Här underlättar kommentarer otroligt mycket.
